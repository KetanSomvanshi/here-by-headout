03

Why this shape

Every dependency was cut until only Python was left. Each choice below is deliberate, and each has a known upgrade path if the demo graduates.

Database

SQLite, not a server

A file in the repo. Zero setup, instant reset (delete the file). Swapping to Postgres is a single connection-string change in SQLModel.

Live

SSE, not Supabase

The dashboard holds one connection; a booking publishes to an in-process bus and fans out. No Redis, no replication — one worker is the trade.

Runtime

One process, no Docker

The entire dependency is “a box that runs Python.” Run it with one command locally; ship it as one process.

04

Ship it two ways

SSE needs a long-lived process, so serverless is out. Both paths below run a normal process — one for the live pitch, one for a durable URL.

For the demo · $0

Local + Cloudflare Tunnel

Run on your laptop, expose a public URL so a phone can scan the QR. Nothing deployed; full control.

uvicorn app.main:app --port 8000 cloudflared tunnel --url http://localhost:8000
For a durable URL

Fly.io + volume

Always-on single machine, SQLite on a mounted volume so data survives restarts. SSE stays connected.

fly volumes create here_data --size 1 fly deploy # auto_stop_machines = false
05

Build is test-driven

Every behavior the prototype faked is pinned as an executable test. Each milestone turns a test group from red to green — “the demo still works” becomes a checkable fact.
